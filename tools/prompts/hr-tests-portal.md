# ПРОМТ · Мокап портала вендора HR-тестирования

Скопируй всё ниже в новую сессию (или используй в этой). Перед запуском приложи логотип клиента, если есть.

---

Ты собираешь **интерактивный демо-мокап** портала для компании, которая **продаёт тесты для найма** (психометрика и ассессменты: HR-отделы покупают тесты, кандидаты проходят их онлайн, заказчик получает отчёты). Мокап — продающая демка для встречи с собственником этой компании: всё должно прокликиваться и объяснять ценность.

## 0. Бренд и стиль — ПЛЯШЕМ ОТ ЛОГОТИПА
- Если приложен логотип: выведи из него палитру, характер шрифтов (гротеск/сериф/округлый), формы (углы/скругления), фирменный мотив. Название компании — из лого.
- Если логотипа нет: рабочее имя **SARAP · оценка кандидатов**; направление «психометрическая лаборатория» — научная точность и доверие: чернильный сине-графитовый + белая бумага + один сигнальный цвет (научный зелёный #1e7d5f для «норма/рекомендован», терракота для рисков). Фирменный мотив — **кривая нормального распределения и процентильные линейки** (гауссиана как декоративный и рабочий элемент шкал).
- Шрифты только с полной кириллицей, НЕ Inter/Nunito. Пара по умолчанию: заголовки **Spectral** (сериф, научность) + текст **Onest** + цифры шкал **JetBrains Mono** (tabular).
- ЗАПРЕЩЕНО: эмодзи вместо иконок (только line-SVG-глифы, точки-статусы, буквенные коды), пилюли-бейджи повсюду, фиолетовые градиенты, каркас «ряд из 4 KPI-карточек + белые скруглённые карточки с тенями». Костяк придумай свой (например: верхняя навигация + рабочая область с линейками, или светлый сайдбар-«оглавление методик»).
- Читаемость важнее красоты: базовый кегль ≥14px, контраст высокий.

## 1. Кто пользуется (роли — переключатель в шапке)
1. **Директор вендора** — вся картина: продажи, клиенты, деньги, методики.
2. **Менеджер продаж** — воронка лидов и клиенты.
3. **HR клиента (кабинет заказчика)** — свои вакансии, кандидаты, отчёты, остаток прохождений. Показать 1 экраном: «так видит ваш клиент».
4. (Не отдельный экран, а мини-мокап в рамке телефона) **Кандидат** — прохождение теста.

## 2. Экраны мокапа (8 шт)

### 2.1 Дашборд вендора
- Метрики: выручка месяца; активных компаний-клиентов; прохождений за месяц; конверсия «пилот → договор»; клиенты с низким остатком кредитов (= сигнал к допродаже).
- Блок «Требует внимания»: у «Kaspi Office» осталось 12 прохождений (был пакет 500) → задача менеджеру; пилот «BI Group» заканчивается через 3 дня; жалоба на опечатку в тесте «Числовой анализ v2».
- Лента событий. Мини-график прохождений по неделям.

### 2.2 Воронка продаж (B2B)
- Канбан: Лид → Демо/пилот → КП отправлено → Договор → Активный клиент.
- Карточки: компания, контакт HR, сколько нанимают в месяц, интересующие тесты, источник (LinkedIn, рекомендация, hh-рассылка, сайт). Пилот = 20 бесплатных прохождений с дедлайном.
- В карточке сделки: переписка WhatsApp/email, задачи, история.

### 2.3 Клиенты (компании)
- Таблица: компания, тариф/пакет, остаток прохождений, израсходовано за месяц, ответственный менеджер, продление.
- Карточка компании: договор, пользователи-HR (кол-во мест), любимые тесты, история закупок, счета, «остаток < 10% — автосчёт на пополнение отправлен».

### 2.4 Каталог тестов (витрина методик)
- Карточки методик по группам: **Способности** (числовой, вербальный, логический анализ), **Личность** (опросник Big Five, мотивация), **Надёжность** (честность/риск-поведение), **Ситуационные SJT** (менеджеры продаж, руководители), **Проф-навыки** (Excel, 1С, языки: рус/каз/англ).
- В карточке: длительность, число вопросов, языки, наличие норм по РК, цена за прохождение, версия и дата ревизии норм, «адаптивный/фиксированный».
- Пометка «собственная разработка / лицензированная методика».

### 2.5 Тестирования (сессии)
- Таблица сессий: кандидат, компания-заказчик, вакансия, батарея тестов, статус (приглашён → начал → завершён → отчёт готов), срок ссылки, флаги честности (переключения вкладок, паузы, слишком быстрые ответы).
- Действия: отправить ссылку в WhatsApp/email, напомнить (авто через 24 ч), продлить срок.
- Групповая выдача: 30 кандидатов на вакансию одним списком.

### 2.6 Отчёт по кандидату (самый продающий экран!)
- Шапка: кандидат, вакансия, батарея, дата, время прохождения, флаги честности = «результату можно доверять».
- **Шкалы с процентилями**: гауссиана с отметкой кандидата, стэны 1–9, сравнение с нормой «специалисты РК» и с профилем вакансии (бенчмарк-полоса).
- Вердикт: «Рекомендован / рекомендован с оговорками / риск» + сильные стороны и зоны риска текстом.
- Сравнение кандидатов: топ-5 на вакансию рядом, ранжирование по соответствию профилю.
- Кнопки: PDF-отчёт заказчику, короткая версия для нанимающего менеджера.

### 2.7 Кабинет HR-клиента («так видит ваш заказчик»)
- Его вакансии, кандидаты со статусами, остаток прохождений крупно, кнопка «пригласить кандидатов», отчёты. Ничего лишнего от вендорской кухни.

### 2.8 Финансы вендора
- Продажи пакетов (100/500/1000 прохождений), списания по клиентам, MRR/повторные закупки, дебиторка, счета. Какие методики зарабатывают больше всего.

### + Рамка телефона (на экране 2.5 или отдельным блоком)
- Прохождение кандидатом: приветствие с лого заказчика, прогресс-бар, таймер, «камера не используется», вопрос с вариантами — аккуратный мобильный интерфейс.

## 3. Данные (вымышленные, реалистичные для Казахстана)
- Клиенты: «Kaspi Office» (условно), «BI Group HR», «Air Astana Recruiting», «Magnum Retail», «Halyk Talent» — 500–1000-пакеты; мелкие: кадровые агентства.
- Кандидаты: казахстанские имена; вакансии: менеджер продаж, бухгалтер, оператор кол-центра, руководитель отдела.
- Цифры согласованные: остатки, списания, выручка сходятся между экранами.

## 4. Технические требования (дом Pllato)
- Один самодостаточный HTML-файл `app/<имя>.html`: vanilla HTML/CSS/JS, без сборки и зависимостей; Google Fonts можно.
- Первой строкой в `<head>`: `<script src="/app/gate.js"></script>` (временный доступ).
- `<meta name="robots" content="noindex,nofollow">`, favicon инлайн-SVG под бренд.
- Верхняя плашка: «Интерактивное демо для <Имени> · все данные вымышленные, всё прокликивается · Pllato».
- Всё кликабельно: строки, карточки, кнопки → тосты, объясняющие ценность функции языком боли клиента (1–2 предложения, продающе, без воды).
- Приветственный тост через 0.7 с: обращение по имени + что посмотреть.
- Адаптив под телефон (мобильное нижнее меню), `?s=<screen>` для прямых ссылок на экран.
- Роли переключаются в шапке и реально фильтруют/меняют данные.

## 5. Definition of done
- 8 экранов + рамка кандидата, все данные сходятся, ни одного эмодзи-иконки, шрифты не Inter/Nunito, костяк не «карточная сетка», гейт работает, на телефоне читается.
