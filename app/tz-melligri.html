1Общие сведения

1.1. Предмет разработки — веб-система (CRM/ERP) для B2B-дистрибуции профессиональной косметологии (далее — Система): продажи препаратов врачам-косметологам через воронки с проверкой диплома, склад с сериями и сроками годности, мульти-прайс, план-факт и KPI-зарплата, каналы Instagram/WhatsApp/Telegram, триггеры, блогер-коды и двусторонняя интеграция с 1С — всё в одном портале.

1.2. Ключевой принцип Заказчика — реальная картина без ручных отчётов: менеджеры вносят данные в одном месте, всё считается само, «не вырежешь, не приклеешь». Собственник видит контролируемые цифры, менеджеров не нужно «дёргать».

1.3. Основание: договор на разработку ПО, КП № MG-2026-01 и настоящее ТЗ. Утверждённый интерактивный макет (демо) — pllato.kz/app/kosmetika/ — является образцом интерфейсов и объёма разработки.

1.4. Стоимость — $2 700 (базовая $3 000 со скидкой 10% за подтверждение в течение 24 часов), срок — 4–6 недель, первая версия ядра — ~7 дней. Оплата этапами: старт $300 → скелет $1 200 (принято) → финал $1 200. В собственность, на сервере Заказчика, без абонплаты за «коробку».

3Функциональные требования — модули

М1

Дашборд собственника

в демо
  • Обращения по дням, выручка, план месяца, средний чек — в реальном времени, без ручных отчётов
  • Лиды Instagram/WhatsApp/Telegram считаются автоматически; топ препаратов и топ клиентов
  • Всё, что собственник хочет контролировать, — на одном экране
М2

Мульти-воронки

в демо
  • Воронки: продажи, повторные клиенты, семинары — со своими этапами; этапы меняются без программиста
  • Перенос сделок между воронками, авто-этапы, канбан по стадиям
  • Обязательные поля — сделку нельзя двигать, пока не заполнено нужное
М3

Диплом-гейт

в демо
  • Цены показываются только врачам: пока диплом не прикреплён — цены скрыты (🔒), сделку нельзя двигать
  • Документы (диплом, сертификаты) тянутся в карточку из Viber/Telegram/Instagram
  • Так конечным потребителям цены не показываются — защита B2B-модели
М4

Карточка клиента

в демо
  • Последние покупки, обязательные поля, документы, статус диплома, канал, менеджер, оборот — всё видно при звонке
  • История заказов и переписки; импорт базы из Google Sheets / 1С
М5

Каналы и лиды (омни-инбокс)

в демо
  • Instagram, WhatsApp и Telegram в одном инбоксе; переписка ведётся из CRM
  • Каждый новый диалог = лид, считается сам; видно, сколько пришло с каждого канала за день/неделю
  • Подключение каналов — около $20 за канал в месяц (оплачивается отдельно)
М6

Заказы и мульти-прайс

в демо
  • Сборка заказа прямо в сделке; нужный прайс (дистрибьютор / опт / розница) подтягивается автоматически
  • На каждую позицию несколько цен — вручную цены не вводятся
  • Согласование и отгрузка заказа; расходная накладная уходит в 1С
М7

Склад дистрибьютора

в демо
  • Номенклатура ~500 SKU; серии/лоты, сроки годности с подсветкой истекающих, остатки и резерв
  • Приходы (с указанием серии/лота и срока) и расходы по заказам; остатки в реальном времени
  • Инвентаризация с обязательными причинами расхождений; отгрузки со списанием серий/лотов
М8

План-факт и KPI → зарплата

в демо
  • План на месяц по каждому менеджеру; система сама считает % выполнения на текущую дату
  • Зарплата по тирам выполнения (до 70% — оклад, 70–90% +3%, 90–110% +6%, 110%+ +10%)
  • KPI: % плана, выручка, кол-во чеков, средний чек, звонки, новые и повторные клиенты — отдельные отчёты не нужны
М9

Триггеры

в демо
  • Дни рождения (поздравление + спецпредложение), повторный заказ (30+ дней тишины → задача «позвонить»)
  • Заход в каталог по cookie без заявки → задача; праздники (авторассылка по сегментам); «диплом не подтверждён» → напоминание менеджеру
  • Триггеры включаются тумблером, тексты сообщений настраиваются под MELLIGRI
М10

Блогеры и промокоды

в демо
  • Врачу/блогеру — персональный код; клиент с кодом получает скидку, заказ закрепляется за кодом
  • Атрибуция: привлечённые клиенты, оборот по коду, расчёт выплаты блогеру — считается само
М11

Звонки и телефония

в демо
  • Журнал звонков, запись и транскрибация разговоров; звонок из карточки, входящий открывает карточку клиента
  • Контроль работы менеджера по скрипту (оценка соблюдения по транскрибации); пропущенный → задача «перезвонить»
М12

Аналитика

в демо
  • По менеджерам, воронкам и каналам; конверсия, LTV, повторные покупки, продажи по городам, топ препаратов
  • Instagram/WhatsApp/Telegram, телефония с записью, сайт с cookie-трекингом — подключаются под процессы
М13

Двусторонняя интеграция с 1С

в демо
  • Работаете в портале — данные синхронны с 1С; в 1С заходить не нужно
  • Продажа = проведение: счёт-фактура и накладная создаются в 1С, остатки зеркалируются в обе стороны
  • Импорт номенклатуры и клиентов; настройка обмена под конфигурацию Заказчика
М14

Кабинет косметолога

в демо
  • Отдельный портал клиента: каталог с ценами (после подтверждения диплома), история заказов, повторный заказ в один клик, статус доставки
  • Доступ по ссылке из WhatsApp, без установки приложения

5Что оплачивается отдельно

Не входит в стоимость разработки и оплачивается отдельно по факту: подключение каналов Instagram/WhatsApp/Telegram (~$20 за канал в месяц); провайдер WhatsApp (~10 000 ₸/мес через партнёра); хостинг — на старте бесплатные тарифы облака, платное — при росте нагрузки; доработки после сдачи — $20/час по согласованной оценке. Абонентской платы за «коробку» нет.
