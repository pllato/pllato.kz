Портал управления сетью детских садов «Зайка»

Базовая разработка (стандартный пакет) · приложение к КП № ZK-2026-01 и договору-оферте
ТЗ № TZ-ZK-2026-01
Заказчик: Екатерина · сеть детских садов «Зайка», Алматы
Исполнитель: ИП «STUDYSTORIES.APP» (Pllato)
Дата: 7 июля 2026 · версия 1.0

1Общие сведения

1.1. Предмет разработки — веб-портал управления сетью из 6 детских садов (далее — Портал): набор детей, коммуникации, дети и группы, табель, обучение, найм и персонал, склад, касса, кабинет родителя, роли и отчётность.

1.2. Портал заменяет текущий контур Заказчика: Битрикс24 (воронка, кол-центр) и Google-таблицы (табель, склад, отчёты) — с миграцией данных.

1.3. Основание: договор-оферта Pllato (pllato.kz/offer), КП № ZK-2026-01, настоящее ТЗ. Утверждённый интерактивный макет (демо) — pllato.kz/app/zaika.html — является образцом интерфейсов и объёма базовой разработки.

1.4. Стоимость базовой разработки — 1 500 000 ₸ (без НДС), срок — 4–6 недель, первая версия ядра — ~7 дней. Оплата: 10% старт → 50% остатка после сдачи ядра → остаток по сдаче. Правки по ядру (1–2 согласующих созвона, ~30% объёма доработок по ходу) входят.

2Роли и права доступа

Воспитательсвоя группа: отметки табеля, занятия, отчёты дня, навыки детей
Завхозсклад: остатки, заявки садиков, закупки, выдача
Бухгалтерсводы табеля, перерасчёты, журнал платежей, сверка кассы (просмотр/подтверждение)

Каждое действие фиксируется в журнале (кто, что, когда) — история не редактируется задним числом.

3Функциональные требования — модули базовой разработки

М1

Воронка набора детей + миграция из Битрикс24

в демо
  • Этапы: новая заявка → экскурсия назначена → пробный день → договор → зачислен; настраиваемые
  • Карточка заявки: ребёнок, возраст, родитель, желаемый садик/группа, источник, переписка, звонки
  • Лист ожидания по группам и садикам; перевод заявки в карточку ребёнка при зачислении
  • Автосообщения: приглашение на экскурсию с адресом/датой, напоминание за 2 часа, после визита — оценка качества 1–5, при оценке 5 — просьба оставить отзыв в 2ГИС
  • Миграция из Битрикс24: контакты, сделки, стадии, история — переносим; SLA на ответ (сигнал при простое заявки)
М2

Изи-бот прогрева заявок с таргета

в демо
  • Сценарий: анкета (возраст ребёнка, район) → серия видео/сообщений с настраиваемыми интервалами → приглашение на экскурсию → передача менеджеру
  • Редактор последовательностей: тексты, видео, интервалы Заказчик меняет сам, без программиста
  • Статус прогрева виден в карточке заявки; аналитика: сколько дошло до менеджера/экскурсии
М3

Единое окно коммуникаций: WhatsApp + Instagram

в демо
  • WhatsApp через Green API (белый канал без пометок), Instagram Direct — в одном окне
  • Переписка хранится в карточках заявок/детей — смена номера или сотрудника не теряет историю
  • Шаблоны ответов, файлы/фото, ответ от имени садика с любого устройства
  • Автонапоминания об оплате должникам со ссылкой на оплату
М4

Телефония (SIPUNI)

в демо
  • Звонок из карточки в 1 клик; входящие определяются и прикрепляются к карточкам
  • Записи разговоров хранятся и прослушиваются в Портале, в карточке ребёнка/заявки
  • Пропущенный звонок автоматически создаёт задачу администратору
М5

Группы и дети

в демо
  • Карточка ребёнка: родители и контакты, аллергии/особенности, посещаемость, оплаты, переписка и звонки, прогресс развития, документы
  • Группы по садикам (ясли → подготовительная), перевод между группами и садиками без потери истории
  • Поиск и фильтры по всей сети (владелица) / своему садику (заведующая)
М6

Табель

в демо
  • Отметки в один клик: был / болел / отпуск — заведующая или воспитатель, с телефона/планшета
  • Свод: прибыло/убыло за месяц, на начало/конец месяца, по садику и по сети
  • Перерасчёт абонплаты по факту посещения — свод для бухгалтера одной кнопкой
  • Питание по фактическим дням; отметки занятий приходящих педагогов → расчёт их оплаты по факту
  • Переносит текущую Google-таблицу табеля целиком
М7

Обучение и развитие

в демо
  • Единая программа сети: методист задаёт план — расписания по каждой группе каждого садика
  • Воспитатель отмечает проведение занятия одним нажатием — контроль выполнения программы
  • Навыки ребёнка: ежемесячные отметки воспитателя, динамика для родителей и владелицы
  • Отчёт дня родителям в WhatsApp с фото — одной кнопкой на группу
М8

Найм (HR-воронка)

в демо
  • Этапы: отклик → собеседование → стажировка (3 дня, оценка заведующей) → оформление
  • Отклики с HeadHunter/Instagram создают карточки автоматически; бот отправляет онлайн-тест в WhatsApp
  • Онлайн-тесты кандидатов (педагогика, безопасность, ситуации) — результат в карточке
  • Массовые WhatsApp-рассылки по откликам; кадровый резерв с тегами
М9

Персонал

в демо
  • Штат по садикам: должности, документы; медкнижки и аттестации с напоминаниями за 30 и 7 дней
  • Пришёл/ушёл: отметка на планшете садика; опоздания, переработки, табель для расчёта зарплат
М10

Склад завхоза

в демо
  • Номенклатура: питание, хозтовары, канцелярия, гигиена; остатки и минимумы
  • Заявки садиков завхозу; закупка одной кнопкой из минимумов и заявок; выдача с фиксацией по садикам
  • Движение по каждой позиции: кто, когда, куда; перенос текущей таблицы завхоза целиком
М11

Касса: первичный приём денег и учёт

в демо
  • Приём оплат на месте: Kaspi / наличные / перевод — заведующая проводит платёж в 3 клика
  • Типы платежей: абонплата (с привязкой к ребёнку и месяцу), вступительный взнос при зачислении, прочие
  • Квитанция родителю в WhatsApp автоматически; долг закрывается сразу
  • Наличные у заведующих: остаток на руках по каждому садику, напоминание о сдаче, подтверждение бухгалтером — сверка «приняла = сдала» сходится сама
  • Журнал платежей без правок задним числом (аудит); оплата по ссылке отмечается без ручной сверки
М12

Веб-кабинет родителя

в демо
  • Доступ по ссылке из WhatsApp, без установки приложения; каждый видит только своего ребёнка и свою группу
  • Расписание группы, отчёты дня с фото, оплата (счёт + ссылка), переписка с садиком внутри кабинета
  • Переписка хранится у Заказчика: увольнение воспитателя не уносит историю и контакты
М13

Задачи, дашборд сети и базовая отчётность

в демо
  • Задачи и поручения с ответственными и сроками; автозадачи от системы (пропущенный звонок, просроченная заявка)
  • Дашборд владелицы: заполняемость по садикам, план/факт абонплаты, долги, набор, найм, сигналы «требуют внимания»
  • Лента событий сети; базовые отчёты: посещаемость, сборы, долги, движение детей

5Не входит в базовую разработку

Зафиксировано на встрече 06.07.2026: база — без финансового контура и без приложения. Отдельными пакетами после сдачи ядра: финансовый контур (дашборды уровня Power BI, P&L, зарплаты — ориентир 0,3–0,4 от базы), интеграция 1С, маркетинг и лояльность (отчёты таргетолога/SMM, акции, промокоды, «приведи друга»), приложение-коммьюнити (видео-эпизоды занятий с камер, опросы/голосования, публичная часть). База проектируется так, чтобы пакеты подключались без переделок.

6Нефункциональные требования

  • Веб-приложение, адаптивное под телефон, планшет и компьютер; современные браузеры
  • Доступ по ролям (раздел 2), журнал действий, защита персональных данных согласно законодательству РК
  • Резервное копирование — ежедневно; хостинг до 10 ГБ — бесплатно; домен ~$10/год
  • Код и данные — собственность Заказчика; никакой абонплаты за пользователей и лицензий
  • Масштабирование: новый садик/город добавляется как филиал без доработок и лицензий
