Управление договорами-офертами

Создавайте новые редакции, замораживайте старые, обновляйте архив. Все изменения генерируются как ZIP-пакет — коммитите в репозиторий pllato/pllato.kz и GitHub Pages развернёт автоматически за 30 секунд.

Как это работает

Для каждого продукта Pllato — отдельная типовая оферта на своём URL. Сейчас опубликовано две: аренда CRM и кастомная разработка. Если запустите новый продукт (например, готовые модули или маркетинговые услуги) — добавьте сюда третью.

Когда меняете условия — создаёте новую редакцию: текущая /offer/{product}/index.html замораживается копией в /offer/{product}/v{дата}/, главная заменяется новым содержанием, архив пересобирается. Старые клиенты остаются на своей редакции до следующего платежа.

Новый продукт-оферта

Создаст новую папку /offer/{slug}/ и стартовую редакцию 1.0. После публикации добавьте оферту в админку (она подтянется автоматически).
URL будет: pllato.kz/offer/{slug}/

Пакет готов · опубликуйте на pllato.kz

ZIP-архив скачан. Внутри — все файлы в правильной структуре. Распакуйте поверх ~/Desktop/Cloude/pllato-kz-mockup/ (или другого клона), затем выполните команды ниже.

Команды для публикации

  1. Скопировать содержимое распакованного ZIP в локальную копию репозитория
  2. Выполнить git-команды ниже
  3. GitHub Pages автоматически развернёт изменения за 30 секунд
cd ~/Desktop/Cloude/pllato-kz-mockup
# Распакуйте ZIP сюда поверх существующих файлов
