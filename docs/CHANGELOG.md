# CHANGELOG

История ревизий Pllato CRM. Свежие сверху.

См. также `app/version.js` (читается в UI → Настройки → О приложении).

---

## v0.11 · rev-10 (2026-05-18)
- `login.html` переведён на Google Identity Services + собственный JWT (`/auth/google` в Worker), Firebase Auth удалён.
- `app.html` переведён с Firebase RTDB на Worker API:
  - пользователи: `/users/list`, `/users/save`, `/users/delete`
  - документы для ACL в админке: `/store/pull` + `/store/push` (`collection=documents`)
- `contact-center.html` переведён на JWT-сессию и Worker API без Firebase токенов.
- `team.html` переведён в безопасный redirect на новую CRM (`/crm/`) вместо аварийного режима Firebase.
- `documents/main.js` переведён на Worker API для users/documents (без RTDB/Firebase Auth).
- `firebase-rules.json` переименован в `firebase-rules.deprecated.json` (историческая справка, больше не используется).

## v0.10 · rev-9 (2026-05-17)
- Документы: вкладка `Калькулятор` теперь открывается первой в builtin-документе «Система мотивации партнёра».
- Документы: добавлено поле `kind` (`builtin | markdown | file | embed`) с авто-миграцией существующих записей.
- Документы: в модалке создания/редактирования добавлен выбор формата:
  - `текст (markdown)`
  - `файл` (загрузка до 50 МБ в Firebase Storage)
  - `внешняя ссылка` (`https://...` с авто-детектом провайдера).
- Документы: рендер по типу контента:
  - `file`: inline preview для PDF/изображений/видео/аудио/текста, карточка-скачивание для остальных типов;
  - `embed`: iframe для Google Docs/Sheets/Slides, Figma, YouTube/Vimeo; кнопка открытия для Notion/other.
- Документы: карточки списка теперь показывают разные иконки/цвета/лейблы для файлов и ссылок (PDF/PPTX/PNG/TEXT/Google Docs и т.д.) как в мокапах.
- Документы: улучшена устойчивость загрузки сотрудников (`/users`) — увеличен таймаут и добавлен повтор без таймаута для селектора доступа.
- Документы: при удалении или замене файлового документа выполняется best-effort очистка старого файла из Storage.
- Документы: исправлено зависание кнопок `Изменить/Поделиться` при медленном чтении `/users` (ограниченное ожидание + открытие модалки без блокировки UI).
- Документы: кнопка `Новый документ` больше не блокируется в fallback-режиме; редактирование доступно сразу.
- Документы: ошибки сохранения в модалке теперь показываются явно (инлайн текст + тост поверх модалки), добавлен таймаут/понятные ошибки upload в Storage.
- Документы: добавлен быстрый health-probe RTDB (`/.json?shallow=true`); при `423 Locked` раздел переходит в read-only без долгого зависания лоадера.
- Документы: при проблемах Firebase используются локальные кэши `users` и `documents` (если есть), чтобы список и «Поделиться» не пустовали после успешных прошлых сессий.
- Документы: create/edit/share/delete и upload теперь явно блокируются в read-only с понятным сообщением сверху, вместо «тихих» зависаний.
- App: при успешном чтении `/users` сохраняется кэш реестра сотрудников для переиспользования в `documents.html`.

## v0.9 · rev-8 (2026-05-16)
- Новый отдельный раздел `documents.html` (вне CRM) с редизайном списка документов.
- Вкладки `Общие` / `Личные`, поиск, создание/редактирование/удаление документов.
- Обновлённая модель `/documents` в RTDB: `authorId`, `scope`, `sharedWith` + миграция legacy-записей.
- Встроенный документ «Система мотивации партнёра»: 10 вкладок, калькулятор без оклада, сценарии.
- Диалог «Поделиться» для автора/админа и управление доступом к документам сотрудника из `app.html`.
- Карточка «Документы» в `App` ведёт на `documents.html` и расположена после «Мигратор Bitrix».
- Добавлена защита от вечной загрузки: таймауты на Firebase `users/documents` и аварийный экран с кнопкой перезагрузки/повторного входа.
- Если `/users` недоступен по timeout, раздел открывается через fallback (кэш пользователя/аварийный режим), чтобы не блокировать работу с документами.
- Убрана блокирующая зависимость от `/users`: документы открываются сразу для авторизованного пользователя, `/users` подгружается фоном для имён и админ-меток.
- Добавлен жёсткий fallback старта: если Firebase/Auth тормозит, страница автоматически открывает встроенный документ в автономном режиме (без зависания лоадера).
- Исправлен баг watchdog: аварийный fallback теперь не сбрасывается раньше времени во время `bootstrapSession`.
- В модалке «Редактировать документ» добавлен выбор сотрудников (из `App` users) с поиском и сохранением `sharedWith`.
