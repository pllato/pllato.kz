# Редизайн карточки сделки (CRM)

Изменены 2 файла: `app/views/deals.js` (+~190 строк, ~410 строк в diff с учётом структурных правок) и `styles.css` (+215 строк в конец файла).

Бэкенд / Firebase / Worker / схема данных — **не трогали**. Новые поля в БД не нужны: лента активности (`deal_activities`) уже существует, заметки и WA-сообщения пишутся туда же.

## Что изменилось в карточке сделки

### Удалено из формы
- Поле **Дедлайн** (`dueDate`) — больше не редактируется и не отображается
- Textarea **Заметки** (`notes`) — заметки делаются через ленту активности (вкладка «Заметка» в таймлайне)
- Иконки коммуникации (📞/💬/✉) из шапки — перенесены в нижнюю панель

**Важно**: legacy-значения `dueDate` и `notes` у существующих сделок **сохраняются нетронутыми** в БД — submit формы больше не пишет эти поля, поэтому старые данные не затираются. Если когда-нибудь захочешь мигрировать legacy notes в активности — отдельной таской.

### Добавлено

**Компактная карточка контакта** вместо typeahead-дропдауна:
- Аватар + имя + телефон + email + компания
- Кнопка ✎ (Изменить) — раскрывает inline-форму правки полей контакта прямо в карточке сделки
- Кнопка 👥 (Сменить) — переключает на typeahead с поиском и созданием нового контакта
- Hidden input `name="contactId"` сохранён — submit формы продолжает работать без изменений

Управляется через `state.contactMode = "view" | "edit" | "change"`.

**Нижняя панель действий** (`.deal-action-bar`) — 3 кнопки:
- **Позвонить** → `openCommunicate({type: "call"})` (как раньше)
- **WhatsApp** (главная, фирменный зелёный) → открывает плавающее окно
- **Заметка** → активирует таб «Заметка» в таймлайне и фокусирует поле ввода

**Плавающее WhatsApp-окно** (`.wa-float`) — fixed позиция снизу справа, `z-index: 200` (выше модалки):
- Аватар + имя + название канала + кнопка закрытия
- Сообщения через `renderDialogMessages` из `wa_dialog.js` (как раньше)
- Compose-форма с текстом + ссылкой на файл + чекбоксом voice
- При отправке: вызов `sendWaFromDialog`, плюс зеркало в `deal_activities` (тип `whatsapp`)
- Sync-цикл `ensureDealChatLoop` обновлён — синкается пока открыто либо модалка, либо float
- Мобильная адаптация: на ширине ≤640px окно во всю ширину экрана

## Что НЕ изменилось

- Таймлайн активности: все 5 типов (Заметка / Письмо / Дело / WhatsApp / Звонок) — как и просил
- Стейдж-бар сверху карточки
- Drag&drop канбана
- Поиск по CRM
- Структура `state` (расширили двумя полями: `waFloat`, `contactMode`)
- API стора, формат записей, экспорты модуля
- `wa_dialog.js`, `communicate.js`, `typeahead.js`, `store.js` — не правились

## Известные мелочи

1. **Создание нового контакта через «Сменить»**: работает через существующий `attachTypeahead` с `onCreate` — создаёт пустой контакт с только именем и телефоном. Чтобы дозаполнить остальные поля — кнопка ✎ (Изменить) после привязки.
2. **Action bar только у существующих сделок** (`!isNew`): для новой сделки сначала сохрани, потом появятся действия. Связано с тем, что без сохранённого `contactId` нечего звонить/писать.
3. **`renderDealChatPane` удалена** — функцию вычистил вместе с её CSS-классом `.deal-chat-pane`, который использовался только внутри модалки сделки. Стили оставил в файле как мёртвый код (стереть отдельной правкой если хочется чистоты).
